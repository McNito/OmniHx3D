About/Walkthrough

What is OmniHx3D?


OmniHx3D is a fork of BabylonHx, which is an ultra-portable lightweight 3D engine.
It is a port of the excellent BabylonJS 3D engine to the Haxe programming language.

This Haxe port allows the Babylon 3D engine to run natively on all platforms, running on one of multiple Haxe graphics backends.

OmniHx3D is compatible with three major Haxe libraries/frameworks for cross-platform development and graphics backends: Lime/OpenFL, NME and Snow.
With OmniHx3D, you can create native games for these platforms and more: Windows, Mac, Linux, iOS, Android, BlackBerry, FirefoxOS, Tizen, HTML5..., all from a single code base!


OmniHx3D is especially flexible when it comes to building for the web.There are three ways to build for the web:

  • You can use it in your Lime/OpenFl or Snow project and build for JS/HTML5 just like for any other target supported by these graphics frameworks.
  • When targeting the web only, OmniHx3D can work with no dependency on external libraries at all.
    Simply create Haxe JS project and build it by passing "purejs" parameter to Haxe compiler.
  • Finally, you can compile the OmniHx3D engine to a JS file and then use it from JavaScript in exactly the same way you would use the original BabylonJS engine. In fact, you can simply replace the original babylon.js file in an existing BabylonJS javascript project with the babylon.js file compiled from OmniHx3D and everything will work perfectly.



Why OmniHx3D?


The primary reason for doing this port was a practical need for a cross-platform 3D engine. A true 'code once, run everywhere' solution - achieved with OmniHx3D

Another reason was was curiosity. Not to see if it can be done but too see how far I can go with Haxe. It turns out that I can go as far as I wish.
With Haxe, there are no limits!

OmniHx3D is WIP, its not 'feature complete' (compared to the original JS version) but is very close. It will likely cover your current needs; however, some elements are still missing (or need to be fixed), especially on C++ targets.



How do I get started?


To get started working with OmniHx3D, first you need to install the Haxe compiler and NekoVM. That will also install the Haxe library manager "haxelib" which you will use to install the necessary libraries/frameworks.

Then you'll have to install runtime files for C++ backend for Haxe - HXCPP, execute from command line: 

haxelib install hxcpp


After that, you can choose one of the three fantastic Haxe libraries/frameworks:

Lime/OpenFL http://www.openfl.org/
NME http://nmehost.com/nme/
Snow http://snowkit.org/

or you can install all of them. OmniHx3D is compatible with all three.

Personally, I recommend Lime/OpenFl.

To start with Lime execute from the command line: 

haxelib install lime

Next, set up lime: 

haxelib run lime setup


When Lime is installed and configured you will need to install the build tools for each platform you wish to build OmniHx3D for; if you want to build for Windows you need to run: 

haxelib run lime setup windows

This will set up Visual Studio and the Microsoft Build tools. Each platform has a similar set up process; for example, for Android: 

haxelib run lime setup android

This command will download the Android SDK and NDK and Ant (and Java JDK if not present).
You should accept the default locations that the installer will offer.
For the web (JS/HTML5) target, no additional software is required.


Finally, you can install OmniHx3D with the following command, which will also install dependecies actuate, poly2trix. It will also offer to install lime: 

haxelib install OmniHx3D

You now have the tools required to run and create OmniHx3D projects.

Sample project/template


To run the sample project and get a template, download the sample project repository from https://github.com/exaphaser/OmniHx3D-WhatCubes and navigate to the root of the repository.
Then run from the command line: 

haxelib run lime test project.xml html5

This should build a sample called "WhatCubes" sample and run it in your browser.


You can also build for Windows/OS X by changing the build target (you need a Mac to build for Mac or iOS): 

haxelib run lime test project.xml windows
haxelib run lime test project.xml mac


Note that the WhatCubes demo includes cursor locking, which does not work on mobile; however, you can remove this code and run WhatCubes on Android/iOS with: 

haxelib run lime test project.xml android
haxelib run lime test project.xml ios


If you wish, you can also install NME and Snow: 

haxelib install nme
haxelib git snow https://github.com/underscorediscovery/snow.git

For Snow you'll also need to install Flow which is used to build Snow projects:

haxelib git flow https://github.com/underscorediscovery/flow.git

You should now be ready to start with OmniHx3D!
